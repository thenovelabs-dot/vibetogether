# 같이바코할사람 — PRD (1차 MVP)

> 서비스명: **같이바코할사람**
> 플랫폼: **웹앱 (Vite + React)** · 인증: **Google OAuth (Supabase Auth)** · DB: **Supabase** · 알림: **Resend 이메일** · 배포: **Vercel**
> 문서 버전: v1.0 (1차 MVP 반영) · 최종 업데이트: 2026-06-13

---

## 0. 한 줄 요약

바이브코딩하는 사람들이 동네에서 모각작 모임을 열고 참여하는 웹 커뮤니티.
모임 등록 → 댓글/신청으로 사람을 모으고 → 게시판·프로덕트 쇼케이스로 커뮤니티를 키운다.

---

## 1. 배경 & 문제 정의

- AI 코딩 도구로 빠르게 만드는 **바이브코딩**이 확산되면서 혼자 작업하는 1인 개발자·메이커가 늘고 있음
- 혼자 작업은 집중력·동기부여 유지가 어렵고 고립감이 큼. "같이 있되 각자 일하는" 모각작은 집중과 가벼운 네트워킹을 동시에 충족
- 기존 모임 앱은 범용이라 바이브코딩 니치에 안 맞음

**핵심 가설:** "내 동네 + 같은 관심사(바이브코딩)"라는 조건을 좁히면 참여 장벽이 낮아져 모임이 실제로 성사된다.

---

## 2. 타겟 유저

- 바이브코딩 / 사이드프로젝트를 하는 **1인 개발자·디자이너·메이커**
- 카페·코워킹에서 같이 작업할 동료를 찾는 사람
- 1차 시드 지역: **서울 강남·성수·판교 인접** 메이커 밀집지

## 4. 핵심 기능 상세

### 4.1 인증 & 온보딩

- Google OAuth → Supabase Auth (토큰 자동 관리)
- 신규 유저: 닉네임 → AI 도구 선택 → 직업 선택 완료 시 `/meetup` 진입
- 기존 유저: `/meetup`으로 바로 이동
- 비로그인 유저도 콘텐츠 열람 가능. 작성/신청 시도 시 로그인 유도

### 4.2 홈 통합 피드 (`/home`)

- 최신 모임 + 게시판 글 + 프로덕트를 하나의 피드로 혼합 노출 (생성 시각 기준 정렬)
- 우측 패널(데스크톱): 내 신청 현황 + 호스트로서 대기 중인 신청 요약
- Supabase Realtime 구독으로 새 콘텐츠 자동 반영

### 4.3 모임 목록 (`/meetup`)

- 모임만 필터링한 전용 리스트뷰
- **지역 필터 칩**: 전체 / 내 지역 / 추가한 지역 (지역 추가 모달로 검색 후 추가)
- **정렬**: 최신순 / 임박순
- Supabase Realtime 구독

> 지도뷰는 2차 MVP에서 추가한다. 현재는 리스트뷰만 존재.

### 4.4 모임 등록

| 항목 | 필수 |
|---|---|
| 제목 | ✅ |
| 설명 | ⬜ |
| 장소명 | ✅ |
| 날짜·시간 | ✅ |
| 모집 인원 | ✅ |
| 지역 | ✅ |

### 4.5 모임 상세 + 댓글

- 모임 정보 + 자유 댓글/대댓글
- 댓글은 참여 카운트 아님 — 인사, 질문 등 자유 사용
- 모집 인원(`capacity`)은 **목표 표시값** — 실시간 집계·하드 마감 없음
- 댓글 작성 시 → 호스트에게 이메일 알림 (Resend)
- 호스트: 수정/삭제 가능

### 4.6 신청 시스템

```
신청자: 신청하기 → status: pending
호스트: /meetup/:id/applications 에서 수락/거절
수락 → status: accepted + 신청자에게 이메일 (호스트 연락처 포함)
```

- 정원 도달 시 신청 버튼 비활성화
- 신청은 1회만 (UNIQUE 제약)

### 4.7 게시판

- 카테고리: 일반 / 모각작 후기 / 바이브코딩 꿀팁 / 바이브코딩 질문
- 정렬: 최신순 / 인기순
- 댓글/대댓글 지원
- Supabase Realtime으로 새 글 자동 반영

### 4.8 프로덕트 쇼케이스

- 바이브코딩으로 만든 프로젝트/서비스 홍보
- 좋아요·저장 기능
- 태그, 서비스 URL, SNS URL 등록 가능

### 4.9 이메일 알림 (Resend)

| 트리거 | 수신자 | 내용 |
|---|---|---|
| 모임 댓글 | 호스트 | `"'{모임 제목}'에 새 댓글이 달렸어요"` |
| 신청 수락 | 신청자 | `"'{모임 제목}' 참가 신청이 수락됐어요. 호스트 이메일: {email}"` |

## 5. 유저 플로우

### 탐색 → 참여

```
앱 첫 진입
  ↓
Google 로그인 (비로그인이면)
  ↓
신규 유저: 온보딩 (닉네임 → AI 도구 → 직업)
기존 유저: 바로 홈(/home 또는 /meetup)
  ↓
모임 목록(/meetup)에서 관심 모임 발견
  ↓
모임 상세 → 댓글 작성 또는 신청하기
  ↓
(신청 시) 호스트 수락 → 이메일로 호스트 연락처 전달
```

### 모임 개설

```
/meetup 또는 /home에서 FAB "모임 등록하기"
  ↓
제목 / 장소 / 날짜·시간 / 모집 인원 / 지역 입력
  ↓
등록 → 모임 상세 이동 → 리스트에 실시간 반영
```

## 6. 데이터 모델 요약

```
profiles          — 유저 (Google OAuth uid 연결)
meetups           — 모임
meetup_applications — 모임 신청 (pending/accepted/rejected)
comments          — 모임 댓글 (대댓글: parent_id)
board_posts       — 게시판 글
board_comments    — 게시판 댓글 (대댓글: parent_id)
showcase          — 프로덕트
showcase_likes    — 프로덕트 좋아요
showcase_saves    — 프로덕트 저장
notifications     — 앱 내 알림
notices           — 공지사항
```

- RLS 적용 — 본인 데이터만 수정 가능
- `auth.users.email` 직접 참조 (profiles에 email 미저장)

## 7. 비기능 요구사항

- 모바일 우선 반응형 (하단 탭바 ↔ 사이드바 전환)
- Supabase Realtime으로 주요 목록 자동 갱신 (새로고침 없이)
- RLS 필수 — 권한 없는 수정/삭제 방어
- 환경변수: `VITE_SUPABASE_URL`, `VITE_SUPABASE_ANON_KEY`, `RESEND_API_KEY`
