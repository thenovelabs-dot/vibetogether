# 같이바코할사람 — 모각작 모집 서비스 PRD (1차 MVP)

> 서비스명: **같이바코할사람** · 바이브코딩하는 사람들을 위한 동네 기반 모각작 모집 커뮤니티
> 플랫폼: **앱인토스 (토스 미니앱, WebView/Granite)** · 지도: **네이버 지도(웹 SDK)** · 로그인: **토스 로그인** · 알림: **앱인토스 푸시(기능성)**
> 문서 버전: v0.5 (MVP) · 작성일: 2026-06-02

---

## 0. 한 줄 요약

내 주변에서 열리는 **바이브코딩 모각작 모임**을 지도에서 찾고, 누구나 장소·시간·인원을 정해 가볍게 모임을 열면, 참여 희망자가 **댓글로 모이는** 동네 기반 커뮤니티.

---

## 1. 배경 & 문제 정의

- AI 코딩 도구로 빠르게 만드는 **바이브코딩**이 확산되면서 혼자 작업하는 1인 개발자·메이커가 늘고 있음.
- 혼자 작업은 집중력·동기부여 유지가 어렵고 고립감이 큼. "같이 있되 각자 일하는" 모각작은 집중과 가벼운 네트워킹을 동시에 충족.
- 기존 모임 앱(소모임/문토 등)은 범용이라 바이브코딩 니치에 안 맞고, **가까운 동네에서 가볍게 모이는** 흐름에 최적화되어 있지 않음.

**핵심 가설:** "내 동네 + 같은 관심사(바이브코딩)"라는 조건을 좁히면, 참여 장벽이 낮아져 모임이 실제로 성사된다.

---

## 2. 목표

### MVP 목표
1. 내 주변 모각작 모임을 **지도에서 쉽게 발견**한다.
2. 누구나 **장소/시간/인원**만 정하면 모임을 연다.
3. 참여는 **댓글**로 가볍게 이뤄진다.

### 성공의 모습 (정성)
- 모임이 올라오면 댓글로 사람이 모이고, 실제 오프라인 만남으로 이어진다.

---

## 3. 타겟 유저

- 바이브코딩 / 사이드프로젝트를 하는 **1인 개발자·디자이너·메이커**
- 카페·코워킹에서 같이 작업할 동료를 찾는 사람
- 1차 타겟: 20~30대, **서울 — 강남·성수·판교 인접 메이커 밀집지**부터 시드

대표 페르소나
- *"혼자 카페에서 작업하는데 누가 옆에 있으면 더 집중될 것 같은 메이커"*
- *"동네에서 비슷한 거 만드는 사람 만나보고 싶은 개발자"*

---

## 4. MVP 범위

### ✅ In Scope (이번에 만든다)
- 온보딩: **토스 로그인 → 닉네임 + 동네(위치) 등록**
- 홈: **지도 기반 모임 탐색** (+ 리스트 토글)
- 모임 등록: **장소 / 날짜·시간 / 모집 인원**
- 모임 상세 + **댓글 (참여 카운트 없음, 자유 댓글)**
- **호스트 알림**: 내 모임에 새 댓글이 달리면 호스트에게 **기능성 푸시** 1종
- 간단 마이페이지

### ❌ Out of Scope (이번엔 안 한다 → 추후)
- 실시간 채팅 / DM
- 평점·리뷰·매너온도
- 동네 GPS 인증 (당근식 위치 인증)
- 호스트 수락 방식 + 참여자 대상 알림 확장(모임 변경·취소·임박 리마인더) — Phase 2
- 비로그인 열람 (1차는 토스 로그인 후 이용)
- 결제, 정기모임/그룹 관리

> **플랫폼:** **앱인토스(토스 미니앱)** 형태로 토스 앱 안에서 동작한다. 로그인은 **토스 로그인**을 사용하고, 토스 웹뷰 환경 기준으로 **모바일 우선**으로 설계한다.

## 5. 핵심 기능 상세

### 5.1 온보딩
- 진입 시 **토스 로그인** (앱인토스 제공 인증 사용 → 별도 회원가입/비밀번호 없음)
- 로그인 후 **닉네임 입력** — **중복 불가(고유값)**. 이미 사용 중이면 재입력 요청
- **위치 설정**
  - GPS 권한 허용 → 현재 위치 기준 동네 자동 설정
  - 권한 거부/실패 시 **동네 직접 검색·선택** (예: "역삼동")
- 위치 인증은 MVP에서 생략 — 등록한 동네를 그대로 신뢰
- 완료 후 홈(지도)으로 진입

> 토스 로그인으로 본인 인증 부담을 없애고, 추가 입력은 닉네임+동네 한 단계로 끝낸다.

### 5.2 홈 — 모임 탐색 (지도뷰 / 게시판뷰)
- 상단 토글로 **지도뷰 ↔ 게시판뷰** 전환 (기본: 지도뷰)
- **지도뷰**
  - 네이버 지도에 **내 주변 모임 핀** 표시
  - 핀 탭 → 하단 요약 카드(제목 / 장소 / 시간 / 모집 M명)
- **게시판뷰**
  - 모임 글이 **카드 피드로 세로 스크롤** (제목 / 장소 / 시간 / 모집 M명 / 호스트 닉네임 / 거리)
  - 정렬: 최신순 / 임박순 / 거리순
  - 카드 탭 → 모임 상세
- 공통: 상단 "내 동네: OO동" 표시 + 변경, 우측 하단 **+ (모임 만들기)** 버튼

### 5.3 모임 등록
입력 항목:
| 항목 | 필수 | 설명 |
|---|---|---|
| 제목 | ✅ | 예: "강남역 카페 모각작 같이 하실 분" |
| 설명 | ⬜ | 어떤 작업/분위기인지 자유 기술 |
| 장소 | ✅ | 지도에서 핀 찍기 또는 장소 검색 |
| 날짜·시간 | ✅ | 시작 일시 (종료는 선택) |
| 모집 인원 | ✅ | 최대 인원(정원) |

- 등록 시 호스트의 동네 기준으로 지도에 노출
- 등록 후 모임 상세로 이동

### 5.4 모임 상세 + 댓글
- 모임 정보(제목/설명/장소/시간/호스트 닉네임/**모집 M명**) 표시
- **댓글 영역**
  - 댓글은 **자유 댓글** — 참여의사, 질문, 인사 등 무엇이든. **참여 인원으로 카운트하지 않음**
  - 모집 인원은 호스트가 정한 **목표값으로 표시만** (실시간 인원 집계·마감 없음)
  - **새 댓글이 달릴 때마다 호스트에게 푸시 발송** ("댓글이 달렸어요", 아래 5.6 참고)
- 호스트는 본인 모임을 **수정 / 삭제** 가능

> 댓글을 참여로 자동 환산하지 않으므로 모델이 단순하고 정확하다(댓글이 곧 참여 확정은 아니므로). 호스트는 댓글 알림을 보고 직접 분위기를 파악한다. 호스트 수락·정원 관리 같은 정교한 참여 관리는 Phase 2.

### 5.5 마이페이지 (간단)
- **프로필**: 닉네임 / 동네 표시
- **내가 작성한 글**: 내가 만든 모임 목록
- **댓글 단 글**: 내가 댓글 단 모임 목록
- **프로필 수정**: 닉네임·동네 변경 (닉네임 변경 시 중복 검사 재적용)

### 5.6 알림 (앱인토스 푸시)
- **MVP 알림 1종:** 내 모임에 새 댓글이 달리면 → 호스트에게 발송
  - 예시 문구: "[같이바코할사람] '{모임 제목}'에 새 댓글이 달렸어요"
- **종류:** 기능성 메시지 (마케팅 수신 동의 불필요)
- **대상 지정:** 토스 로그인의 `x-toss-user-key` 기준으로 호스트에게 발송
- **채널:** 앱 미실행 시 OS 푸시 + 토스 앱 내 알림함 동시 노출
- **제약:** 메시지 템플릿은 **사전 검수 필요(영업일 약 2~3일)**. 댓글 원문을 그대로 보내지 않고, 검수받은 템플릿에 변수(모임명)만 끼워 발송. → 개발 초기에 템플릿 문구 확정 후 검수 신청 권장
- **수신 거부:** 알림 해제 경로를 명확히 제공 (가이드 권장사항)
- Phase 2 확장: 참여자 대상(모임 변경·취소·시작 임박 리마인더), 호스트 수락 결과 알림

## 6. 핵심 유저 플로우

**탐색 → 참여**
1. 앱 진입 → (최초) 온보딩으로 닉네임·동네 등록
2. 홈 지도에서 주변 모임 핀 확인
3. 마음에 드는 모임 탭 → 상세 진입
4. 댓글 작성 → **호스트에게 "댓글이 달렸어요" 푸시 발송** (참여 카운트 없음)

**개설**
1. 홈 우하단 **+** 버튼
2. 제목·장소·시간·인원 입력 후 등록
3. 모임 상세 생성 → 지도에 노출 → 댓글로 사람이 모임

## 7. 화면 목록 (MVP)

1. 온보딩 (닉네임 + 동네 설정)

3. 모임 상세 (+ 댓글)

5. 마이페이지

## 8. 데이터 모델 (간단)

```
User
- id
- toss_user_key   (토스 로그인 식별자)
- nickname
- region (동네명 또는 코드)
- lat, lng (선택: 동네 중심 좌표)
- created_at

Meetup
- id
- host_id (→ User.id)
- title
- description (nullable)
- place_name
- lat, lng
- start_at
- capacity (정원)
- region
- status (open / closed)
- created_at

Comment  (모임 글의 댓글)
- id
- meetup_id (→ Meetup.id)
- user_id (→ User.id)
- content
- created_at
```

- 댓글은 **참여 인원으로 집계하지 않음** (자유 댓글). 새 댓글 발생 시 호스트에게 푸시.
- `capacity`(모집 인원)는 호스트가 정한 **목표 표시값** — 실시간 카운트·하드 마감 없음.

## 9. 비기능 요구사항

- **앱인토스(토스 웹뷰)** 환경 기준 **모바일 우선** 설계 (한 손 사용 고려)
- **토스 로그인** 연동 — 토스가 제공하는 인증/사용자 식별(`x-toss-user-key`) 사용
- **앱인토스 푸시/알림** 연동 — 기능성 메시지 발송, 템플릿 사전 검수(영업일 2~3일) 일정 반영
- **네이버 지도** SDK 연동 (지도 표시, 핀, 장소 검색)
- 위치 권한 거부 시 fallback(수동 동네 선택) 필수
- 위치·개인정보 등 민감 데이터 보호 (URL/쿼리스트링에 위치값 노출 금지)
- 빠른 로딩 · 가벼운 첫 진입 경험 (웹뷰 특성상 초기 로딩 최적화 중요)

### ⚠️ 착수 전 필수 기술 검증 (스파이크)
- **네이버 지도 타일 CSP 이슈:** 앱인토스 WebView에서 네이버 지도 JS API의 지도 타일 도메인(map.pstatic.net)이 CSP에 막혀 배경이 안 보이는 사례가 보고됨. 타일 도메인 화이트리스트 가능 여부를 토스 측에 확인하고, 안 되면 **대안 지도(예: 카카오맵)** 검토.
- **TDS(토스 디자인 시스템):** 비게임 WebView 미니앱은 TDS가 검수 승인 기준에 포함된다는 문서가 있음 → 지금 설계 단계의 우선순위는 아니지만, **출시 검수 전 가이드에서 재확인** 권장.
- **외부 스킴 호출 제한:** `window.location.href` 외부 스킴 호출은 차단되므로 지도/외부 연동은 웹 SDK·앱인토스 제공 API로 처리.
- **푸시 템플릿 검수 리드타임(2~3 영업일):** 알림 문구 확정 후 일찍 검수 신청.
