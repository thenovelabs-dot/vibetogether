# agent.md — 앱인토스 전용 작업 지침

이 문서는 앱인토스용 빌드/개발을 맡은 AI 코딩 에이전트가 따라야 할 기준이다.
기존 웹 서비스의 디자인은 유지하고, 앱인토스 정책상 위험한 기능만 분리한다.

동반 문서: `CLAUDE.md`, `prd.md`, `design.md`

---

## 목표

기존 웹앱과 같은 Supabase 프로젝트를 사용하되, 앱인토스에서는 다음 범위로 별도 빌드한다.

- 유지: 홈, 게시판, 프로덕트, 검색, 공지, 마이페이지, 약관, 개인정보처리방침, 문의
- 제외: 모임 탭, 모임 상세/등록/수정/신청자 관리, 모임 신청 상태, 커피챗
- 교체: Google OAuth 로그인 → 토스 로그인
- 디자인: 기존 `design.md`의 토큰, 레이아웃, 컴포넌트 감각을 그대로 유지

앱인토스 버전은 "기존 웹앱의 축소판"이 아니라 "토스 미니앱 안에서 완결되는 게시판+프로덕트 커뮤니티"로 다룬다.

---

## 가장 중요한 원칙

### 1. 디자인은 바꾸지 않는다

앱인토스 대응 작업 중 아래를 임의로 변경하지 않는다.

- 색상 토큰: `#ae49fd`, `#fafbfb`, `#f3f4f6`, `#101828` 등
- 카드, 버튼, 입력, 모달의 radius/spacing
- 사이드바/하단 탭바 구조
- `public/icons/*.svg` 기반 아이콘 시스템
- Pretendard 기반 타이포그래피와 기존 화면 밀도

앱인토스 정책 대응은 기능/라우팅/인증/링크 처리로 해결한다. 디자인을 TDS 스타일로 새로 갈아엎지 않는다.

### 2. 웹 빌드와 앱인토스 빌드는 분리한다

웹 버전 기능을 삭제하지 않는다. 앱인토스 전용 분기는 환경변수나 빌드 타깃으로 처리한다.

권장 환경변수:

```env
VITE_APP_TARGET=web
VITE_APP_TARGET=apps-in-toss
```

권장 헬퍼:

```ts
export const APP_TARGET = import.meta.env.VITE_APP_TARGET ?? "web";
export const isAppsInToss = APP_TARGET === "apps-in-toss";
```

권장 스크립트:

```json
{
  "build": "vite build",
  "build:ait": "VITE_APP_TARGET=apps-in-toss vite build"
}
```

macOS/Linux 이외 환경까지 고려해야 하면 `cross-env` 도입을 검토한다.

### 3. 앱인토스에서 위험한 기능은 숨김이 아니라 접근 차단한다

탭에서만 제거하면 안 된다. 다음 경로는 앱인토스 타깃에서 라우트 자체를 막거나 `/home`으로 리다이렉트한다.

- `/meetup`
- `/meetup/:id`
- `/meetup/new`
- `/meetup/:id/edit`
- `/meetup/:id/applications`

또한 다음 UI도 앱인토스 타깃에서 제거한다.

- 모임 탭
- "모임 등록하기" FAB
- 홈 피드 내 모임 카드
- 마이페이지의 내 모임/신청 현황
- 알림 드롭다운의 모임 관련 이동
- 커피챗 신청/수락/프로필 연결 CTA

## 앱인토스 정책 기준

### 로그인

앱인토스 미니앱 내 로그인은 토스 로그인만 사용한다. Google OAuth, Kakao, Naver, Supabase OAuth 버튼은 앱인토스 타깃에서 노출하지 않는다.

현재 웹 버전:

- `src/screens/LoginScreen.tsx`
- `src/contexts/UserContext.tsx`
- Supabase Google OAuth

앱인토스 버전에서 해야 할 일:

- 로그인 버튼 문구를 토스 로그인 기준으로 변경
- 앱인토스 토스 로그인 API 연동
- 서버 간 연동이 필요한 경우 mTLS 지원 서버/Edge Function 구조 검토
- Supabase `users.id`와 토스 사용자 식별자를 어떻게 매핑할지 명확히 구현

토스 로그인 연동 전에는 앱인토스 출시 가능 빌드로 보지 않는다.

### 외부 링크

앱인토스는 외부 링크와 자사 앱 설치 유도를 제한한다.

앱인토스 타깃에서는 다음을 기본값으로 둔다.

- 앱 설치 유도 문구 금지
- 앱마켓 링크 금지
- 외부 결제창 이동 금지
- 프로덕트의 `service_url`, `sns_url`은 핵심 CTA가 아니라 보조 정보로 취급
- 외부 링크를 열어야 하면 앱인토스 SDK의 외부 URL 열기 방식을 사용하고, 링크 목적을 명확히 제한

프로덕트 상세의 핵심 경험은 미니앱 안에서 제목, 설명, 이미지, 태그, 댓글로 완결되어야 한다.

### 채팅/만남/소셜 리스크

앱인토스 타깃 1차 버전에서는 아래를 비활성화한다.

- 오프라인 모임 생성/참여
- 모르는 사람에게 커피챗 신청
- 공개 프로필에서 직접 연결을 유도하는 CTA
- 위치 기반 사람/모임 탐색

이 기능을 앱인토스에 다시 넣으려면 신고, 차단, 운영정책, 모니터링, 제재, 성인/본인인증 필요 여부를 먼저 검토한다.

### 사용자 생성 콘텐츠

게시판/댓글/프로덕트는 UGC이므로 앱인토스 타깃에서 최소한 아래 운영 기능을 갖춘다.

- 게시글/댓글 신고 기능
- 프로덕트 신고 기능
- 문의/고객센터 진입점
- 약관 및 개인정보처리방침
- 탈퇴 및 개인정보 삭제 경로
- 관리자 삭제/제재 운영 정책

## 구현 가이드

### 추천 파일 구조

앱 타깃 분기는 한 곳에 모은다.

```text
src/config/appTarget.ts
src/config/features.ts
```

예시:

```ts
export const features = {
  meetups: !isAppsInToss,
  coffeechat: !isAppsInToss,
  googleLogin: !isAppsInToss,
  tossLogin: isAppsInToss,
  externalProductLinks: !isAppsInToss,
};
```

각 화면에서 `import.meta.env`를 직접 반복해서 읽지 않는다.

### 라우팅

`src/App.tsx`에서 앱인토스 타깃일 때 모임 라우트를 등록하지 않거나, 명시적으로 리다이렉트한다.

권장:

- 웹 타깃: 기존 라우트 유지
- 앱인토스 타깃: `/meetup*` → `/home`

### Layout

`src/components/Layout.tsx`에서 `MAIN_TABS`, `NEW_ACTION`, `ALL_TAB_PATHS`를 feature flag로 필터링한다.

앱인토스 타깃의 탭:

- 홈
- 게시판
- 프로덕트
- 마이페이지

홈의 기본 FAB는 앱인토스에서 모임 등록이 아니라 숨기는 것을 기본으로 한다. 홈에서 글쓰기 CTA를 새로 만들고 싶으면 디자인을 유지한 채 별도 논의 후 추가한다.

### Home / Feed

앱인토스 타깃에서 홈 피드는 모임 데이터를 불러오지 않는다.

권장 구성:

- 최신 게시글
- 최신 프로덕트
- 공지 또는 운영 안내

Supabase Realtime 구독도 앱인토스 타깃에서는 `meetups` 채널을 만들지 않는다.

### MyPage

앱인토스 타깃에서 제거:

- 내 모임
- 신청한 모임
- 호스트 신청 관리
- 커피챗 받은/보낸 요청

유지:

- 내 게시글
- 내 프로덕트
- 프로필 수정
- 문의
- 약관/개인정보
- 로그아웃/탈퇴

### 알림

앱인토스 타깃에서 모임/신청/커피챗 링크는 만들지 않는다.
기존 DB에 해당 알림이 남아 있더라도 클릭 시 안전하게 `/home` 또는 관련 없는 화면으로 리다이렉트한다.

## Supabase 사용 기준

동일 Supabase 프로젝트를 사용해도 된다. 단, 앱인토스 사용자를 기존 Google OAuth 사용자와 섞을 때 다음을 명확히 해야 한다.

- 토스 로그인 식별자와 Supabase Auth user id의 매핑 방식
- 기존 `users` 테이블의 `id`, `email`, `nickname` 처리
- 토스 로그인에서 email을 받을 수 없는 경우의 온보딩 흐름
- RLS 정책이 앱인토스 인증 구조에서도 안전하게 동작하는지

앱인토스에서 Supabase anon key를 쓰는 것은 가능하지만, service role key는 클라이언트에 절대 노출하지 않는다.

## 배포 전 체크리스트

- `npm run build`가 웹 타깃에서 성공한다.
- 앱인토스 타깃 빌드 스크립트가 별도로 존재한다.
- 앱인토스 타깃에서 모임/커피챗 라우트가 직접 URL 접근으로도 열리지 않는다.
- 앱인토스 타깃에서 Google 로그인 버튼이 노출되지 않는다.
- 앱인토스 타깃에서 외부 앱 설치 유도/앱마켓 링크/외부 결제 링크가 없다.
- 게시판/댓글/프로덕트에 신고 또는 문의 진입점이 있다.
- 약관, 개인정보처리방침, 탈퇴 경로가 접근 가능하다.
- 기존 웹 타깃의 디자인과 주요 기능이 깨지지 않는다.
- 모바일 WebView 높이, safe-area, 하단 탭바 겹침을 실제 기기 또는 샌드박스 앱에서 확인한다.

---

## 금지

- 앱인토스 대응을 이유로 전체 디자인을 새로 만들지 않는다.
- 기존 웹 버전에서 모임 기능을 삭제하지 않는다.
- 앱인토스 타깃에서 Google OAuth를 남겨두지 않는다.
- 모임탭만 숨기고 `/meetup` 라우트를 방치하지 않는다.
- 외부 링크를 핵심 플로우로 만들지 않는다.
- service role key, mTLS 인증서, 비밀키를 프론트엔드 번들에 넣지 않는다.

---

## 작업 우선순위

1. `VITE_APP_TARGET` 기반 앱 타깃/feature flag 추가
2. Layout 탭/FAB 분리
3. `/meetup*` 및 커피챗 접근 차단
4. Home/MyPage/Notification에서 모임·커피챗 데이터 제거
5. 앱인토스 로그인 설계 및 토스 로그인 연동
6. 외부 링크 정책 정리
7. 신고/운영/탈퇴 동선 보강
8. 앱인토스 샌드박스에서 QA
